◈ WHAT IS SOVEREIGN QRNG?

▼

Your computer is turning physical noise from real hardware into cryptographic-grade random numbers — right now, with no cloud connection required.

Most software uses mathematical formulas to fake randomness. If an attacker knows the formula, they can predict every number it generates. SOVEREIGN QRNG measures real physical events that not even the laws of physics allow anyone to predict in advance.

NEAR QUANTUM
Your webcam, radio dongle, CPU clock jitter, and microphone generate random thermal noise continuously. We harvest entropy from all four simultaneously and mix them together.
TRUE QUANTUM
A laser pointed at a USB webcam generates photon shot noise — randomness from quantum mechanics itself. The exact arrival time of each photon is fundamentally unpredictable, not just hard to predict.

◈ WHY YOUR RANDOMNESS MATTERS

▼

Every encrypted connection you make — HTTPS, SSH, VPN — starts by generating a random secret key. If that number can be predicted, your encryption is worthless. An attacker who guesses your key can decrypt everything, including past traffic they recorded.

✓Quantum entropy — Even with infinite computing power, the next number is physically impossible to predict. Proven by quantum mechanics.
~OS entropy (os.urandom) — Good in practice. Relies on hardware event collection. Can be weak on fresh systems or embedded devices with little activity.
✗Math-based PRNG (rand(), Math.random()) — Never use for security. Entirely predictable from any known output.
✗Cloud RNG APIs — You are trusting the provider completely. A backdoored provider gives you fake randomness and you cannot tell.

◈ HOW YOUR SOURCES WORK

▼
WEBCAM · CAM-A

Your camera sensor contains millions of light-detecting wells. Even in darkness, stray photons and electron thermal motion create random noise in every pixel. We grab the least-significant bit of each colour channel — the noisiest part — and extract the randomness from the difference between frames.

~3,000 bit/s · always available · no extra hardware
RTL-SDR RADIO

A $25 USB radio dongle contains an ADC that samples the air 256,000 times per second. Tuned to a quiet frequency, all it picks up is thermal Johnson noise — the random motion of electrons in the antenna. Impossible to predict or reproduce.

~8,000 bit/s · best near-quantum source available
CPU JITTER

Your processor's clock is not perfectly stable. Cache misses, interrupt handling, and pipeline stalls cause sub-nanosecond timing variations that are unpredictable. We measure these with a high-resolution timer and XOR with Windows CNG (BCryptGenRandom).

~8,000 bit/s · pure software · no hardware needed
MICROPHONE ADC

Your microphone amplifier generates thermal noise even in silence — electrons jiggling due to heat. At 44.1 kHz, the least-significant bits of every audio sample contain this pure physical noise. Slow but always present.

~700 bit/s · slow but always available
LASER CAM · TRUE QUANTUM

Photons from a laser arrive at the camera sensor one by one. Quantum mechanics makes their exact arrival times fundamentally unknowable — not just hard to predict, but provably impossible by the laws of physics. This is Poisson shot noise: the purest randomness in the universe.

~6,000 bit/s · TRUE quantum mechanical
ESCHER-6 CASCADE

All sources are mixed through 6 passes: Fold → Rotate → Fall → Braid → Scatter → SHA-256 Seal. This whitens any bias (making the output perfectly uniform) and ensures no single weak source can compromise the final output.

output: cryptographic-grade uniform bytes

◈ QUICK START

▼

INSTALL

  1. Install Python 3.10+ from python.org
  2. Run: pip install Pillow opencv-python sounddevice pyaudio
  3. Double-click NEAR QUANTUM.lnk on Desktop
  4. Dashboard opens at http://127.0.0.1:8888/near

ADD RTL-SDR RADIO DONGLE

  1. Plug in the RTL-SDR USB dongle
  2. Download and run Zadig from zadig.akeo.ie
  3. In Zadig: Options → List All Devices
  4. Select Bulk-In, Interface (Interface 0)
  5. Choose driver: libusbK — NOT WinUSB — then click Install Driver
  6. NEAR engine detects the dongle within 10 seconds automatically

ADD LASER CAM (TRUE QUANTUM)

  1. Point a laser pointer at a USB webcam lens in a dark room
  2. Plug in the USB webcam — it appears as CAM-B in the TRUE QUANTUM tab
  3. Run python true_entropy.py to start the TRUE engine

AI ADVISOR (AUTOMATIC HELP)

  1. Install Ollama from ollama.com
  2. Run in a terminal: ollama pull gemma2:2b
  3. The SYSTEM ADVISOR panel activates automatically — no other setup needed
  4. When anything goes wrong, the advisor diagnoses it and shows fix steps instantly

◈ TROUBLESHOOTING

▼
RTL-SDR shows OFFLINE
  1. Check USB cable and try a different port
  2. Open Zadig → install libusbK driver (not WinUSB)
  3. Disable USB selective suspend in Device Manager
  4. Restart near_entropy.py
RTL-SDR drops every ~10 seconds
  1. This is Windows USB selective suspend cutting power
  2. Device Manager → Universal Serial Bus → USB Root Hub
  3. Properties → Power Management → uncheck "Allow computer to turn off this device"
  4. Repeat for every root hub listed
Camera shows OFFLINE
  1. Verify webcam_server.py is running on port 8090
  2. Try replugging the USB camera
  3. Run: python webcam_server.py
Audio shows OFFLINE
  1. Windows Settings → Sound → check microphone is not muted
  2. Windows Settings → Privacy → Microphone → allow app access
  3. Restart near_entropy.py
Entropy rate is very low
  1. Verify all 4 sources show LIVE in the NEAR QUANTUM tab
  2. RTL-SDR gain should be 29 dB — check near_entropy.py
  3. Move SDR antenna away from USB 3.0 cables (cause interference)
Vault is not filling
  1. Check C:\QRNG_Pool\ has free disk space
  2. Verify near_entropy.py is running on port 8766
  3. Check near_offset.txt contains a plain number (not corrupted)
